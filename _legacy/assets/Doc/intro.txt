Descrição Técnica: Arquitetura e Funcionamento da Base de Conhecimento
1. Visão Geral da Arquitetura
O projeto é uma Single Page Application (SPA) de front-end, executada inteiramente no lado do cliente (navegador). Sua arquitetura é projetada para ser "serverless", ou seja, não depende de um back-end ou de um banco de dados externo. A aplicação carrega uma estrutura HTML inicial (index.html) e, a partir dela, todo o conteúdo e as transições de tela são gerenciados dinamicamente via JavaScript, proporcionando uma experiência de usuário fluida e sem recarregamentos de página.

O estado da aplicação, como o módulo atualmente visualizado ou o tema selecionado, é gerenciado por meio de variáveis JavaScript e persistido no localStorage do navegador.

2. Tecnologias e Bibliotecas Empregadas
Linguagens Base:

HTML5: Utilizado para a estrutura semântica inicial da página (index.html).

CSS3: Empregado para estilização. A abordagem é modular, com a folha de estilo principal (style.css) importando outras folhas específicas para a sidebar, responsividade e busca. O sistema de temas (claro/escuro) é implementado de forma eficiente com o uso extensivo de Variáveis CSS (Custom Properties).

JavaScript (ES6+): É a linguagem central que governa toda a lógica, interatividade e manipulação do DOM da aplicação.

Bibliotecas JavaScript:

Fuse.js: Biblioteca de busca "fuzzy" (inteligente) utilizada para implementar a funcionalidade de pesquisa. Ela é importada via CDN no index.html e configurada no script.js.

Locomotive Scroll: Biblioteca para rolagem suave e com efeitos, melhorando a experiência de navegação. Também importada via CDN.

Utilitários CSS:

TailwindCSS: Utilizado via CDN para aplicar classes de utilidade diretamente no HTML, agilizando a estilização de componentes, grids e layout responsivo.

3. Estrutura de Arquivos e Dados
A organização do projeto é modular e segue uma clara separação de interesses.

index.html: Ponto de entrada único que contém a estrutura base (<aside id="sidebar">, <main id="app">, <footer id="footer-search">) e importa todos os arquivos CSS e JS necessários.

assets/css/: Contém as folhas de estilo.

style.css: Define as variáveis globais de tema, tipografia e estilos base.

sidebar.css e responsividade.css: Gerenciam os múltiplos estados da barra lateral e adaptam o layout para diferentes resoluções de tela, respectivamente.

busca.css: Estiliza os resultados da busca, como o destaque (<mark>) em termos pesquisados.

assets/js/: Contém a lógica e os dados.

data/modules/*.js (ex: formatacao.js): Atuam como um banco de dados "hard-coded". Cada arquivo define um módulo de conteúdo (um objeto JavaScript com id, title, icon e um array de topics).

data/index.js: Agrega todos os módulos importados em um único array, forumData, que serve como a fonte de dados primária para a aplicação.

sidebar.js: Contém a lógica encapsulada para gerenciar os estados da barra lateral (expandido, recolhido, flutuante) e seus eventos.

script.js: O orquestrador principal da aplicação. Inicializa todas as bibliotecas, gerencia o estado global, contém as funções de renderização (renderCategories, renderTopics, renderArticle) e manipula os eventos do usuário.

4. Fluxo de Execução e Renderização
O navegador carrega index.html, que por sua vez carrega os assets CSS e JS.

O evento DOMContentLoaded no script.js dispara a inicialização da aplicação.

A função initializeFuse() processa o array forumData para criar um índice de busca otimizado.

A função renderCategories() é chamada, gerando dinamicamente o HTML para os cartões de módulos e injetando-o no elemento <main id="app">.

Qualquer interação do usuário (ex: um clique) aciona uma função de evento. Essa função, por sua vez, chama uma função de renderização apropriada (ex: renderTopics).

A função de renderização apaga o conteúdo existente do elemento #app e o substitui pelo novo HTML gerado, criando a ilusão de uma transição de página.

5. Detalhes de Funcionalidades Chave
Sistema de Busca:

A busca é gerenciada pela biblioteca Fuse.js.

A configuração (searchOptions) está definida para buscar nos campos title (peso 0.6), description (peso 0.3) e group (peso 0.2). O corpo principal do artigo (content) não é incluído na busca por padrão.

Um debounce é aplicado ao evento keyup do campo de busca para otimizar a performance, evitando que a busca seja executada a cada tecla pressionada.

A busca pode ser filtrada para abranger "Todos os Módulos" ou um módulo específico selecionado pelo usuário.

Gerenciamento de Estado da Sidebar:

O estado da sidebar é controlado pela adição/remoção de classes no elemento <body> (sidebar-estado-expandido, sidebar-estado-recolhido, sidebar-estado-flutuante).

Regras de CSS complexas em sidebar.css e responsividade.css usam essas classes para alterar visualmente a largura, a opacidade e a posição dos elementos da sidebar.

O estado da sidebar e o estado de recolhimento da navegação são persistidos entre as sessões usando localStorage.setItem().

Tematização (Theming):

O sistema utiliza variáveis CSS declaradas no seletor :root para o tema claro.

Quando a classe .dark-mode é aplicada ao <body>, um novo conjunto de variáveis CSS é ativado para sobrescrever os padrões e aplicar o tema escuro.

A preferência de tema do usuário também é salva no localStorage.
